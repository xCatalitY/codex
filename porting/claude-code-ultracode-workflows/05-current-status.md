- `/effort ultracode` sets `xhigh` plus `WorkflowMode::Ultracode`.
- `/effort max` is rejected with guidance to use `xhigh` or `ultracode`.
- `/workflow on|off|status|ultracode` exists as the compact workflow mode/status surface.
- `[workflows]` config can set default mode, directories, yield time, and output tokens.
- Workflow mode injects bounded dynamic workflow/ultracode guidance.
- The `workflow` tool is registered when workflow mode is active and uses the code-mode runtime.
- Inline/path/named workflow source resolution exists for `.codex/workflows` and global Codex workflow dirs.
- Live smoke evidence from prior work: inline workflow execution and `tools.exec_command` side effects work when scripts use the current tool namespace.
- Workflow scripts now require Claude-style `export const meta = { name, description, ... }`.
- The metadata block is stripped before execution so scripts remain executable inside Codex's async wrapper.
- The validator rejects missing metadata, non-literal metadata patterns, import/export declarations in the body, dynamic import/require/eval/Function/WebAssembly, and nondeterministic `Date.now`, `Math.random`, and `new Date` body calls.
- The validator now rejects forbidden calls even when whitespace or comments appear before the call paren or around member separators, e.g. `eval /*...*/ (...)`, `require (...)`, `import (...)`, `Date.now ()`, `Date /*...*/ . now ()`, `Math.random ()`, `Math /*...*/ . random ()`, `new /*...*/ Date ()`, and property-access forms such as `globalThis.eval (...)`, while still ignoring matching text inside strings and comments.
- Workflow metadata parsing and JS scanner validation now live in a private `workflow::metadata` submodule, reducing the main workflow handler surface while preserving behavior.
- Workflow run-store and artifact persistence now live in a private `workflow::run_store` submodule, covering run snapshots, status structs, resume cache checks, artifact paths, snapshot/transcript writes, active-run markers, output previews, and run time helpers.
